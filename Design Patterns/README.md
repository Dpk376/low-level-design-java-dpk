# Design Patterns

This directory contains implementations of various Design Patterns in Java.

## Patterns Covered:
1. **Singleton Pattern**: Restricts the instantiation of a class to one "single" instance.
2. **Builder Pattern**: Separates the construction of a complex object from its representation.
3. **Factory Design Pattern**: Provides an interface for creating objects in a superclass, but allows subclasses to alter the type of objects that will be created.
4. **Adapter and Facade Pattern**: 
   - Adapter allows incompatible interfaces to work together.
   - Facade provides a simplified interface to a library, a framework, or any other complex set of classes.
5. **Decorator and Flyweight Pattern**:
   - Decorator lets you attach new behaviors to objects by placing these objects inside special wrapper objects.
   - Flyweight lets you fit more objects into the available amount of RAM by sharing common parts of state.
6. **Prototype and Registry Pattern**:
   - Prototype lets you copy existing objects without making your code dependent on their classes.
   - Registry keeps track of available prototypes.

Each directory contains a README with specific details and tests ensuring their correctness.
